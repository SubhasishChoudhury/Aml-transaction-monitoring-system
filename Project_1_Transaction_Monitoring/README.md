# AML Transaction Monitoring & Risk Scoring System

## Project Overview

This project simulates an Anti-Money Laundering (AML) transaction monitoring system using transaction-level data. The system identifies suspicious transactions, assigns risk scores, generates alerts, and provides analytical insights through exploratory data analysis.

## Business Problem

Financial institutions process thousands of transactions daily. AML teams must identify potentially suspicious transactions and prioritize them for investigation. This project demonstrates a rule-based transaction monitoring and risk scoring framework.

## Technologies Used

- Python
- Pandas
- NumPy
- Matplotlib
- Kaggle Notebook

## Features

### Transaction Monitoring
- Monitors customer transactions.
- Identifies suspicious transaction patterns.

### Risk Scoring Engine
- Assigns risk scores based on transaction characteristics.
- Categorizes transactions into risk levels.

### Alert Generation
- Flags suspicious transactions.
- Generates AML alerts for further investigation.

### Dashboard Metrics
- Total Transactions
- Fraudulent Transactions
- Risk Distribution
- Alert Distribution

## Exploratory Data Analysis (EDA)

The project includes:

- Transaction Type Distribution
- Fraud vs Non-Fraud Analysis
- Risk Level Distribution
- Suspicious Alert Distribution
- Summary Metrics Dashboard

## Key Outcomes

- Processed 50,000 transactions.
- Identified suspicious transactions using rule-based logic.
- Generated AML alerts based on risk scores.
- Categorized transactions into risk levels.
- Produced management-ready AML monitoring insights.
